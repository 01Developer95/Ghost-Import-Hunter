# Ghost Import Hunter v2.0 Roadmap

This document outlines the planned features and improvements for the v2.0 release of Ghost Import Hunter.

## 1. Core Engine Improvements

### üß† Deep Export Analysis (Parser-Based)
*   **Current Limit:** Uses naive Regex which fails on `export * from`, CommonJS, and complex re-exports.
*   **v2.0 Upgrade:** Switch to using the **TypeScript Compiler API** or **SWC** to correctly build a symbol table for every file. This will eliminate false positives with:
    *   Wildcard re-exports (`export * from './utils'`)
    *   CommonJS `module.exports` patterns
    *   Dynamic imports

### ‚ö° Performance & Caching
*   **Async Processing:** Move from synchronous `fs` calls to fully asynchronous/parallel processing.
*   **Incremental Scans:** Cache scan results (hashes of files) to only re-scan changed files, significantly speeding up pre-commit hooks.

### üì¶ Monorepo & Alias Support
*   **Problem:** Path aliases (e.g., `@components/Button`) and Workspaces often confuse the current resolver.
*   **v2.0 Upgrade:** First-class support for `tsconfig.json` `paths` and package managers (npm/yarn/pnpm/bun) workspaces.

## 2. New Features

### üõ†Ô∏è Auto-Fix (`--fix`)
*   Automatically remove unused imports.
*   Suggest removing "ghost" imports that don't exist.
*   (Optional) Attempt to install missing packages if they are available in the registry.

### üôà Granular Ignore System
*   Support for inline comments to ignore specific lines or files:
    ```typescript
    import { weirdThing } from 'legacy-lib'; // ghost-ignore-line
    ```
*   Support for `.ghostignore` file (git-ignore style).

### ü§ñ Interactive Mode
*   Run `npx ghost-hunter --interactive` to walk through errors one-by-one and choose an action:
    *   [Delete Import]
    *   [Ignore]
    *   [Install Package]
